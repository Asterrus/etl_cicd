# Data Dictionary

Описание полей целевых таблиц DWH (схема `dwh`).

---

## dwh.Customer_Dim

Измерение клиентов с историей изменений (SCD Type 2).
Каждое изменение атрибутов клиента создаёт новую запись; предыдущая закрывается.

| Поле | Тип | Описание |
|------|-----|----------|
| `customer_sk` | UUID | Суррогатный ключ. Уникальный идентификатор версии записи в DWH |
| `customer_id` | UUID | Бизнес-ключ. ID клиента из источника (`source.Customers`) |
| `name` | VARCHAR(255) | Полное имя клиента |
| `email` | VARCHAR(255) | Электронная почта |
| `phone` | VARCHAR(20) | Номер телефона. Может быть NULL |
| `valid_from` | TIMESTAMP | Момент, с которого актуальна данная версия записи |
| `valid_to` | TIMESTAMP | Момент закрытия версии. NULL означает, что запись актуальна |
| `is_current` | BOOLEAN | `true` — текущая версия клиента, `false` — историческая |
| `attr_hash` | TEXT | MD5-хэш атрибутов (name + email + phone). Используется для обнаружения изменений |

**Бизнес-правила:**
- На каждый `customer_id` всегда ровно одна запись с `is_current = true`
- При изменении любого атрибута: старая запись получает `valid_to = NOW()`, `is_current = false`; создаётся новая с `valid_from = NOW()`, `valid_to = NULL`, `is_current = true`

## dwh.Product_Dim

Измерение продуктов с историей изменений (SCD Type 2). Структура аналогична `Customer_Dim`.

| Поле | Тип | Описание |
|------|-----|----------|
| `product_sk` | UUID | Суррогатный ключ. Уникальный идентификатор версии записи в DWH |
| `product_id` | UUID | Бизнес-ключ. ID продукта из источника (`source.Products`) |
| `product_name` | VARCHAR(255) | Название продукта |
| `category` | VARCHAR(100) | Категория продукта. Может быть NULL |
| `valid_from` | TIMESTAMP | Момент, с которого актуальна данная версия записи |
| `valid_to` | TIMESTAMP | Момент закрытия версии. NULL означает, что запись актуальна |
| `is_current` | BOOLEAN | `true` — текущая версия продукта, `false` — историческая |
| `attr_hash` | TEXT | MD5-хэш атрибутов (product_name + category). Используется для обнаружения изменений |

## dwh.Sales_Fact

Таблица фактов продаж. Хранит каждую продажу со ссылками на версии измерений,
актуальные на момент совершения сделки.

| Поле | Тип | Описание |
|------|-----|----------|
| `sale_sk` | UUID | Суррогатный ключ записи в DWH |
| `sale_id` | UUID | Бизнес-ключ. ID продажи из источника (`source.Sales`) |
| `product_sk` | UUID | FK → `dwh.Product_Dim`. Версия продукта, актуальная на дату продажи |
| `customer_sk` | UUID | FK → `dwh.Customer_Dim`. Версия клиента, актуальная на дату продажи |
| `sale_date` | TIMESTAMP | Дата и время совершения продажи |
| `amount` | DECIMAL(10,2) | Сумма продажи в рублях |
| `quantity` | INTEGER | Количество единиц товара |

**Бизнес-правила:**
- Дубли не загружаются: повторный запуск ETL не создаёт дублирующих записей
- `product_sk` и `customer_sk` указывают на версию измерения, действовавшую в момент `sale_date` (не текущую)

## dwh.data_quality_checks

Журнал результатов автоматических проверок качества данных.
Записи добавляются после каждого запуска ETL и не перезаписываются.

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | SERIAL | Автоинкрементный первичный ключ |
| `check_name` | VARCHAR(100) | Название проверки: `freshness` или `completeness` |
| `status` | VARCHAR(10) | Результат: `OK` или `FAIL` |
| `value` | DECIMAL(10,4) | Числовое значение метрики (см. ниже) |
| `checked_at` | TIMESTAMP | Момент выполнения проверки (DEFAULT NOW()) |

**Проверки и интерпретация `value`:**

| `check_name` | `value` | Порог | Логика |
|---|---|---|---|
| `freshness` | Часов с последней загрузки | < 24 ч → OK | `NOW() - MAX(valid_from)` из `Customer_Dim` |
| `completeness` | Доля расхождения | ≤ 5% → OK | `ABS(source.Sales - dwh.Sales_Fact) / source.Sales` |
